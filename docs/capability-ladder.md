# Ingot 分析能力阶梯

本文定义 Ingot 从“生产记录可用”到“能够给出工艺优化建议”的长期阶段目标。能力等级按具体的工艺问题评定，不给整个工厂只标一个等级。同一套系统可能在模具寿命分析上达到 L3，在另一项质量问题上只有 L1。

## 数据范围

- 分析输入只保留与工艺过程和质量结果直接相关的核心参数，例如温度、压力、速度、进给、位置、时间、模具或刀具状态。
- 设备、工件、生产周期和工艺阶段编号只用于把记录正确对应起来，不作为工艺参数。
- 不引入现场没有的管理信息，也不按班次、组织或人员表现分析工艺问题。
- 不推测缺失字段。缺少核心参数时，系统应说明缺少什么，并停留在当前可达到的等级。
- 每项工艺只配置实际需要的参数，不建立面向所有工厂的固定字段清单。

## 能力等级

| 等级 | 阶段目标 | 系统可以做什么 |
|---|---|---|
| **L0 生产记录可用** | 核心参数和工艺对象对应正确 | 检查缺失、迟到、采集中断和单位冲突；关联设备、工件、生产周期与工艺阶段；区分机器采集、人工录入和系统估算 |
| **L1 生产过程看得清** | 回答生产过程中发生了什么 | 展示阶段时长、核心参数趋势与分布、异常周期，以及工艺版本或设备之间的差异 |
| **L2 参数关系找得到** | 缩小问题排查范围 | 分析核心参数与质量结果的关系、分层对比、变化幅度、重点排查参数，以及同时变化的其他核心参数 |
| **L3 质量风险可预判** | 在不良发生前提示风险 | 预测不良风险、趋势漂移、模具或刀具寿命和风险时间范围，同时说明预测误差与适用范围 |
| **L4 参数影响可确认** | 确认哪些参数变化确实产生影响 | 分析历史参数调整、受控前后对比、DOE 和现场试验，给出影响方向、影响大小及适用工艺范围 |
| **L5 工艺优化可建议** | 给出工程师可以评审的优化方案 | 在质量、节拍、能耗、设备能力和安全范围内推荐参数区间，并说明预期改善、风险和恢复条件 |

## 升级规则

- 等级不能只按功能是否存在来判断，必须看该工艺问题是否有足够且稳定的生产记录。
- L2 只能列出可能原因和重点排查参数，不能直接写成已经确认的根因。
- L3 必须持续检查预测误差；表现不稳定时退回 L2，只展示参数关系。
- L4 需要受控调整、DOE、现场试验或条件清楚的历史调整记录。普通历史相关关系仍停留在 L2。
- L5 给出参数范围，不给脱离现场约束的单一数值；建议由工程师确认，并通过企业现有系统执行。Ingot 不直接控制设备。

## 接入新工厂时如何使用

接入新工厂时，系统先整理可用的核心参数、单位、生产周期、工艺阶段和质量结果，再由工艺人员确认对应关系。确认结果保存为可复用配置。新增工艺应以配置为主，尽量不修改通用代码；如果关键参数尚未接入，就明确降低当前可提供的分析等级。

本文件说明长期要达到什么能力；当前先做什么、按什么顺序实施，见[实施路线](roadmap.md)。

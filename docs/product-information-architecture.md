# Ingot 产品信息架构

Ingot 的界面围绕“理解运行、定位问题、分析工艺、维护语义”组织，而不是围绕数据库表、API 或内部服务组织。平台不复制 MES 的排产、库存、人员和工单执行能力，也不把原始时序数据作为孤立的数据点展示。

## 设计原则

1. **工作优先**：默认进入工作台，先呈现正在运行、待检、数据异常和接入状态。AI 助手是一级能力，但不是产品首页。
2. **对象优先**：设备和逻辑运行对象是数据入口。用户从对象进入当前上下文、运行记录、事件、质量和关联关系，不需要反复输入对象编号。
3. **上下文固化**：运行记录固化当时生效的产品、配方、工装和质量方案引用；历史分析不依赖后来被修改的主数据。
4. **质量是一等数据**：人工检验、视觉结果、测量值、原始图片和复核记录与过程数据共同进入分析上下文。
5. **周期与连续运行并存**：有周期的设备按生产周期组织；连续运行设备按时间窗口、状态段或事件段组织。二者共享运行对象、事件和分析语义。
6. **配置集中，运行简洁**：日常页面不暴露配置表单。创建和维护从台账、版本目录或任务上下文进入，并提供完整的新增、维护、停用和允许条件下的删除闭环。
7. **状态代替说明**：页面通过标题、字段、状态、空状态和有效操作表达语义。成功使用短暂反馈；常驻提示只用于错误、阻断和风险。
8. **运维隔离**：平台指标和日志属于系统管理；原始指标端点不进入普通用户主导航。

导航采用成熟工业数据平台常见的两级结构：顶部主菜单负责在工作台、AI、运行、质量、分析、数据资产、工装和系统管理之间切换；左侧只展示当前业务域的二级入口。工作台和 AI 助手不显示左侧导航，从而获得完整工作空间。页面标题位于内容工作区，全局搜索固定在顶部。这样既保留 TDengine IDMP 的“全局菜单 + 上下文侧栏”优势，也避免把数据库对象术语直接复制到 Ingot。

## 页面模板

- **工作台**：关键状态、待处理事项、当前生产配置和最近运行记录。
- **对象目录**：左侧对象列表，右侧对象上下文；运行记录、质量、事件和关系不再成为孤岛。
- **查询页面**：紧凑过滤器、结果表格、右侧详情抽屉。
- **任务页面**：任务队列为主，处理表单在抽屉中打开。
- **配置页面**：版本/对象目录在左，详情与编辑在右；新建和维护不得长期占用浏览页面的主要空间。
- **分析页面**：数据范围与对齐条件在上，图表和结论使用完整内容宽度。
- **历史对比**：用户先限定产品系列，再多选运行记录并从已选记录指定基准；先用数据项矩阵展示横向差异，再展示周期上下文明细。系统不得擅自用“最近 N 个”替代用户选择。

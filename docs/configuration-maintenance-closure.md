# 配置创建与维护闭环

Ingot 的用户界面遵循一条统一规则：只要平台允许人工创建一个数据主体，就必须在同一业务菜单中提供查询和维护入口。创建表单不能成为孤立入口。

不同数据语义采用不同维护方式：

- 主数据（组件类型、组件、工装身份）提供列表、详情、维护和停用/退役状态。稳定标识在维护时不可修改。
- 版本化配置（工艺数据模型、配方、分析方案、检测定义、质量方案、工装类型和工装组合版本）保留历史版本。草稿可修改和删除；已发布版本不可覆盖，只能停用或基于原版本创建新版本。
- 现场事实（装模记录、生产上下文、检测记录、附件和复核）采用追加式记录。平台提供历史查询、详情、结束区间、复核和审计，不允许覆盖已经发生的事实。
- 设备事件和采样数据由接入接口写入，通过事件、周期和数据质量页面查询，不提供人工修改入口。

页面验收至少覆盖“创建后能找到、能打开、能执行符合语义的维护动作、历史引用不丢失”四项。

正式产品页面只从空白表单或已保存版本开始维护，不提供“载入示例”“加载演示数据”等入口。行业示例仅保留在开发文档和独立导入工具中，不得伪装成生产主数据。

界面按任务语义选择布局，而不是把所有表单放在页面中央：版本化配置采用“左侧版本目录、右侧详情/编辑”的工作台；主数据维护以列表为主并提供明确的新增、编辑和删除动作；现场录入以待办任务为入口，在抽屉或聚焦区域完成。Ingot 品牌位置使用项目自有标志，通用操作继续使用含义明确的功能图标。

业务页面不提供常驻“刷新”按钮。查询页由条件查询驱动，保存操作完成后自动同步；节点、指标、日志和事件等动态页面采用自动更新或可暂停的实时追踪，并显示更新时间或追踪状态。

正常页面不显示教学式提示条。页面语义由标题、字段、分组、状态和空状态表达；成功操作使用短暂反馈；只有错误、阻断条件或需要用户处理的风险才使用常驻提示。

平台指标可视化页面使用 `/platform-metrics`；`/metrics` 专门保留为 Prometheus 原始指标端点，避免产品页面与机器接口路由冲突。

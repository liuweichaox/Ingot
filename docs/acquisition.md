# 可配置采集

Ingot 把采集建模为版本化的采集任务，而不是把设备协议写进工艺模型。任务发布到指定采集节点后，由边缘端加载并运行；边缘 outbox 负责断网续传，平台保留任务版本、运行状态和标准事件。

## 配置边界

- 工艺数据模型定义稳定数据项、目标数据类型和单位。
- 采集任务定义协议连接、认证凭据引用、采样方式、点位映射、倍率/偏移、数据对象和事件来源。
- 凭据值不进入平台数据库。页面只保存 `env:变量名` 引用，实际值在对应边缘节点的环境中配置。
- 一个任务只使用一种协议；同一设备可以发布多个任务，将遥测、配方或其他主题分别接入同一数据对象。
- HTTP、MQTT 和 Modbus TCP 可以选择源数据时间或边缘接收时间；Modbus 源时间使用可配置的寄存器选择器。OPC UA 使用节点源时间。
- 连续设备不配置运行边界。离散周期设备配置关联号、步序键和边界事件类型；关联号变化时边缘端生成 `cycle.completed` / `cycle.started`，步序变化时生成 `recipe.step_changed`。
- 四种协议都可以配置运行上下文和配方采集。HTTP/MQTT 使用 JSON 路径，OPC UA 使用 NodeId，Modbus 标量上下文使用 `area:address:type` 选择器。

## 协议

| 协议 | 执行方式 | 可配置连接 | 点位选择 |
| --- | --- | --- | --- |
| HTTP 轮询 | 按周期读取 JSON 快照 | URL、接口路径、周期、超时、重连 | JSON 字段路径 |
| MQTT | 订阅后按消息生成一组采样值 | Broker、端口、3.1.1/5.0、客户端 ID、用户凭据、TLS、主题、QoS、保活和会话 | JSON 字段路径 |
| OPC UA | 订阅节点并按发布周期合并为一组采样值 | 端点、安全模式/策略、匿名/用户/证书认证、证书、发布和采样周期 | NodeId |
| Modbus TCP | 合并同一区域的点位后批量读取并生成一组采样值 | 主机、端口、Unit ID、周期、超时、重连 | 寄存器区、地址、源类型、数量、字节序和字序 |

四种协议输出相同的 `ProductionEvent`。倍率和偏移只做确定性的采集换算；工艺含义、质量标签和分析选择仍由各自的一等配置与记录表达。

可运行的光学玻璃模压多协议样例位于 `tools/Ingot.OpticalMoldingSimulator`。它使用同一设备状态同时提供四种服务，并通过发布同一采集任务的新版本演示协议切换。

## 与 TDengine 的关系

页面结构参考了 TDengine 的数据源/任务/Agent、协议连接、数据点选择、映射和运行状态分层，但 Ingot 不复刻其实体定义。Ingot 的目标是形成面向工艺分析的标准事件和制造上下文，因此采集任务直接引用工艺数据模型，并由边缘 outbox 提供断网续传。

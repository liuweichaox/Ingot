# ADR-0003：先进技术的价值放大路线

**状态（Status）：** Proposed
**日期（Date）：** 2026-07-20
**决策者（Deciders）：** Ingot 维护者（@liuweichaox）
**前提：** 技术实现不构成瓶颈。本文只讨论"投在哪里回报最大"，不讨论"能不能做"。

---

## 1. 排序原则

Ingot 的价值是一个**乘积**，不是加和：

```text
价值 = 事实覆盖率 × 推理可信度 × 触达人数
```

三项都是乘数，任何一项趋近 0，另外两项再高也归零。按代码走查的现状估计：

| 因子 | 现状 | 判断依据 |
|---|---|---|
| 事实覆盖率 | **低** | 机器侧靠客户自写适配器（无模板）；人工侧完全为零（见 ADR-0002） |
| 推理可信度 | **中高** | `IPlanValidator` 白名单 + `IEvidenceVerifier` 数字接地 + 有界讨论，方向正确 |
| 触达人数 | **低** | 只有一个 Chat 界面，且要求使用者知道该问什么 |

**结论：先进技术应该优先投在事实覆盖率和触达，而不是继续加深推理。** 一个更聪明的 agent 架在空数据库上，价值仍然是零；而"把车间里的纸和 Excel 自动变成事实"哪怕用很朴素的手段，也是纯增益。

这一条会推翻很多直觉排序 —— 比如"上更强的推理模型"其实排在很后面。

## 2. 架构已经给出了正确的接入方式

代码走查发现一个关键事实：**所有新能力都应该以 `IAnalysisTool` 的形式进入，而不是塞进 prompt。**

```text
IAnalysisTool
├─ Definition.Surface   ← 产品面隔离
├─ Definition.Purpose   ← 只读语义
├─ Definition.InputSchema ← DefaultPlanValidator 逐参数校验
└─ ExecuteAsync → AnalysisToolResult { Evidence[] }  ← IEvidenceVerifier 数字接地
```

任何注册进来的能力，自动继承：权限隔离、参数校验、证据链、可审计、数字必须来自工具结果。这是这套架构最值钱的部分 —— **它把"加能力"和"降可信度"解耦了**。

> 因此本文的每一项，最终都落成一句话："往工具注册表里加什么工具"，或"加第几个 Surface"。凡是需要靠加长 system prompt 来实现的能力，都应该被怀疑。

**当前限制**：`DefaultPlanValidator.TryValidate` 里硬编码了

```csharp
if (!string.Equals(surface, ProductSurfaces.Chat, StringComparison.Ordinal))
    return error = "计划包含无效的产品面";
```

Surface 抽象存在，但只有一个取值。第 3.1 节的常驻代理需要的第一件事，就是让它真正多值化。

## 3. 第一波：放大"事实覆盖率"（最高杠杆）

### 3.1 常驻监测代理 —— 从"你来问"到"它来说"

**这是整份路线里回报最高、且技术最现成的一项。**

现状是纯 pull：必须有人打开 Chat 并且知道该问什么。而工厂里最有价值的那些发现，恰恰是**没有人想到要去问**的那些。

```text
新增 Surface: ProductSurfaces.Monitor
  └─ 订阅已有的 /api/v1/events/stream（SSE 已实现，含 Last-Event-ID 续传）
       └─ 窗口触发 → 复用 check_data_quality / 统计工具
            └─ 命中 → 走已有 webhook 通道推给人
                 └─ 人点进来 = 一个预填好上下文的 Chat run
```

需要的新东西其实很少：第二个 Surface、一套触发规则、一份"已通知过"的去重记忆。SSE、工具、证据链、webhook 全部已存在。

**为什么这是杠杆点**：它把产品从"工具"变成"同事"。工具的使用频次取决于用户的主动性，同事的价值每天自动发生。同时它绕开了"用户不知道该问什么"这个 LLM 产品的通用死穴。

**代价**：误报是这类系统的唯一死因。必须从极高阈值起步，并且每条通知都带"这条有用/没用"的反馈按钮，用反馈调阈值。宁可漏报十条，不可误报一条 —— 第三次误报之后没有人会再点开。

### 3.2 多模态摄取：把纸、Excel 和仪表盘变成事实

ADR-0002 说"真正的对手是 Excel"。与其劝人别用 Excel，不如**直接把 Excel 和纸单吃进来**。

| 输入 | 手段 | 输出 |
|---|---|---|
| 手写检测单、巡检表拍照 | VLM 结构化抽取 → 映射到 `InspectionDefinition` | `InspectionRecord`（`SubmitterVerified=false`，标注 `source=ocr`） |
| 车间历史 Excel / CSV（列名千奇百怪） | LLM 推断列语义 → 生成映射草案 → 人确认一次 → 存为可复用映射 | 批量 `InspectionRecord` |
| 非联网仪表、老设备显示屏拍照 | VLM 读数 | `ProductionEvent` |
| 超差工件照片 | VLM 缺陷描述，作为证据的可检索摘要 | `InspectionEvidenceRef` + 语义索引 |

**关键约束**：抽取结果必须**降级标记**，不能与人工确认的事实混为一谈。建议给 `InspectionRecord` 加 `provenance` 字段（`human` / `instrument` / `extracted`）并在 Chat 回答里显式区分。让 AI 抽取的数字冒充人工确认的数字，会一次性摧毁整个平台的可信度定位。

**为什么值得做**：这是唯一能让客户在**第一周**就看到满库数据的路径。适配器要写几周，纸单拍照当天就有。冷启动问题的答案在这里。

### 3.3 时序基础模型：解决"没有标注数据"

工业 AI 的通用死结是冷启动 —— 新客户没有标注过的异常样本，监督学习无从谈起。时序基础模型（Chronos / TimesFM / Moment 一类）的零样本能力正好对上：

- **零样本异常检测**：新设备接入当天就能给出"这段周期不像它自己"的判断，不需要训练
- **周期向量化 + 相似检索**：`get_cycle_trace` 的上位能力 —— "这个周期和历史上哪几次最像？那几次后来怎么了？" 这是工艺工程师真正的思考方式
- **短期预测**：预测的价值不在准确率，在于给 3.1 的监测代理提供"偏离预期"的触发信号

以 `IAnalysisTool` 形式注册（`find_similar_cycles`、`detect_regime_change`），推理服务独立部署，Central 只调用。

## 4. 第二波：放大"推理可信度"

### 4.1 给 agent 配统计工具箱，而不是让它讲故事

`BoundedInvestigationWorkflow` 的自我声明很诚实：

> "深度分析的多角色讨论只生成候选解释，不能替代确定性统计检验或工程师确认。"

这句 limitation 指出了下一步该做什么 —— **把确定性统计检验做成工具，让讨论的每个假设都能被当场检验**：

| 工具 | 回答的问题 |
|---|---|
| `detect_changepoint` | 这个指标是什么时候开始变的？（变点检测，不是肉眼看趋势） |
| `compute_spc` | Cpk / 控制图 / 西电规则 —— 质量部门唯一认的语言 |
| `compare_cohorts` | 换料前后 / 白夜班 / 不同模具，差异是否显著（含效应量，不只 p 值） |
| `rank_correlates` | 哪些工艺参数与该缺陷共变，**并显式列出可能的混杂因素** |

多角色讨论目前是"三个模型基于同一批工具结果互相辩论"。加上这些工具后，可以升级为**假设 → 检验 → 修正**的闭环：challenge 角色不再只是嘴上质疑，而是能要求跑一次对照检验。

这是"看起来像分析"和"质量工程师愿意签字"之间的分界线。

### 4.2 工艺文档检索：把数据异常和规定对上

facts 回答"发生了什么"，但判断"这是否正常"需要 SOP、工艺规程、设备手册、历史 8D 报告。

```text
retrieve_process_spec(工序/设备/特性)
  → "SOP-041 要求预热 ≥30 分钟"
     配合 get_cycle_trace 的事实 "实际 12 分钟"
        → 从「数据描述」跨到「符合性判断」
```

历史 8D / 纠正措施报告是最珍贵的语料 —— 它们是这家工厂过去所有问题的因果标注集，且外人拿不到。这是客户越用越难迁走的地方。

### 4.3 语义层：让多跳推理成为可能

当前事实是行。加一层轻本体（asset ↔ 工序 ↔ 产品族 ↔ 特性 ↔ 工艺参数 ↔ 物料批次），agent 才能走多跳："这批不良的共同点是都用了 3 号炉 + B 批次原料"。

注意这与 ADR-0002 的 `InspectionDefinition` 注册表是同一件事的两个侧面 —— **先做 `InspectionDefinition`，它是语义层的第一块砖**，不要另起炉灶。

### 4.4 评估集：演进能力的前提

一份 golden question 集（问题 → 期望的工具调用序列 → 期望结论 / 期望的 limitation），跑在 CI 里。

这不性感，但没有它，换模型、改 prompt、加工具都是盲飞 —— 而上面每一项都在改这三样。**评估集不是质量措施，是迭代速度的前提条件。** 现有测试覆盖契约与 token 层，agent 行为层是空的。

### 4.5 长期记忆

工厂的价值在时间里累积："这台设备去年冬天有过类似问题，当时结论是冷却水温"。把每次调查的结论（经工程师确认后）沉淀为可检索记忆，让第 100 次调查比第 1 次聪明。

**必须是"经确认后"沉淀**。未经确认的 AI 猜测进入记忆库，会形成自我强化的错误回路 —— 明年 agent 会引用今年自己的臆测作为证据。

## 5. 第三波：放大"触达"

### 5.1 把 Ingot 做成 MCP Server（分发杠杆，非能力）

事实层 + 只读工具，正好是 MCP 的形状。暴露成 MCP server 之后：

- 客户在自己的 Claude / IDE / 内部 agent 里直接查工厂事实
- Ingot 不必赢得"谁家 Chat 界面更好"这场必输的仗，改为占据"谁家事实层更干净"这个位置
- 触达人数从"愿意打开 Ingot Web 的人"扩大到"已经在用 AI 助手的所有人"

工程量极小（工具注册表已经是标准形状），杠杆极大。**这可能是整份文档里投入产出比最高的一项。**

### 5.2 语音：工位上的真实交互形态

戴手套、油污的手、双手占用 —— 这是键盘触屏解决不了的物理现实。语音提交检测（"三号炉，工件 WP-231，硬度 58.2"）+ 语音提问，是工位场景的正确形态，也是 ADR-0002 §3.0"比 Excel 更快"基准线的终极答案。

前提是 3.2 的定义驱动结构化已经就位 —— 语音只是把结构化槽位填满的另一种输入法，不是自由文本。

### 5.3 主动简报

日报 / 班报由 3.1 的监测代理自动生成：昨天的数据完整性、异常、趋势、待确认项。触达那些永远不会主动打开系统的管理者，同时反向暴露数据缺口（"12 号线昨天没有任何数据"）来倒逼接入。

## 6. 明确不做 / 谨慎做

| 方向 | 判断 |
|---|---|
| **把 LLM 放进控制回路（写 PLC、自动调参）** | **绝不**。这是 Ingot 全部安全叙事和可审计性的基石，破坏它等于变成一家完全不同的、需要承担安全责任的公司 |
| 微调专用工业模型 | 现阶段错误。数据量不够、维护成本高、锁死在一个基座上。工具 + 检索能拿到 90% 的收益 |
| 数字孪生 / 物理信息神经网络 | 单点场景（如热处理炉）有真实价值，但通用平台层做不了。定位成"客户可插入的第三方工具"，不自研 |
| 放开多智能体的轮数和自由度 | 当前 3 轮 9 turn 的硬边界是特性不是限制。放开 = 成本失控 + 不可复现 + 不可审计 |
| 追求"全自动根因分析" | 越过了产品定位。`BoundedInvestigationWorkflow` 只产出候选解释是对的 —— 保留工程师作为最终判断者，既是技术诚实，也是责任边界 |
| 端到端 Text-to-SQL | 已经被现有架构正确拒绝（工具白名单）。任何"让模型更灵活地查数据"的提议都是在开这个口子 |

## 7. 建议顺序

顺序的依据是「解除的阻断」而非「技术先进程度」：

| 波次 | 内容 | 解除什么 |
|---|---|---|
| **W0** | ADR-0002 的 P0（证据上传 / 定义注册表 / 录入页） | 没有它，下面全部架空 |
| **W1** | ① MCP Server ② 多模态摄取（Excel + 拍照） | 触达与事实覆盖，两项最低因子，且工程量都不大 |
| **W2** | ③ 常驻监测代理（第二个 Surface） ④ 评估集 | 产品形态从工具变同事；同时建立迭代地基 |
| **W3** | ⑤ 统计工具箱 ⑥ 工艺文档检索 | 推理从"像分析"到"可签字" |
| **W4** | ⑦ 时序基础模型 ⑧ 语义层 ⑨ 长期记忆 | 复合能力，需要前面的数据量做支撑 |
| **W5** | ⑩ 语音 ⑪ 主动简报 | 体验放大，依赖前面全部就位 |

W1 的两项可以并行且互不依赖，是最快能被外部感知到变化的组合。

---

## 8. 一句话总结

**不要把先进技术用来让 Chat 更聪明，用来让事实更全、让更多人碰得到。** 推理能力的上限由基座模型给定，你追不上也不必追；而"这家工厂的事实有多完整、多可信"是只有 Ingot 能建的护城河 —— 那才是所有先进技术应该落地的地方。
